## Branching & Workflow

- **`main`** is protected. No direct pushes.
- Each team works on feature branches scoped to their project: `case-a/feature-name`, `case-b/feature-name`, etc.
- All merges to `main` go through Pull Requests with at least one review.
- Stay in your own `projects/case-x/` directory. Don't touch other teams' code.

## Tech Stack

- **Languages:** Python, JavaScript/HTML/CSS
- **Tools:** Cursor, GitHub, Canvas LMS API, Google Workspace APIs
- **AI:** Claude, various LLM integrations as needed

## Getting Started

1. Clone the repo: `git clone https://github.com/CaydenAuyang/Civic-AI-Github-Repository.git`
2. Navigate to your project folder: `cd projects/case-x-your-project/`
3. Read your project's README for setup instructions.
4. Create a feature branch: `git checkout -b case-x/your-feature`
5. Code, commit, push, and open a PR.
